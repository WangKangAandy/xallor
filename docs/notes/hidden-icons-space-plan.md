# 隐藏空间实现计划（v1 可用版）

本文档定义「隐藏空间」功能的 v1 落地方案，目标是先做出可用闭环，再为后续云同步与安全增强预留扩展点。  
产品与设置内的展示名称统一为 **「隐藏空间」**（不称「隐藏图标空间」）。

---

## 1. 目标与边界

### 目标

- 支持在图标/文件夹右键菜单中执行“隐藏”。
- 设置 `隐私与安全` 中提供「隐藏空间」开关；开启时必须先完成密码设置。
- 未开启时，**仅显示开关一行**；不显示「浏览隐藏空间」、开发项「重置隐藏文件夹提示」及下方隐藏空间面板。
- 开启后可在设置中进入隐藏空间；进入前需输入密码。
- 隐藏空间内采用**与文件夹内部同款**的图标网格浏览；**浏览态 / 编辑态**切换后再做多选与批量操作。
- 极简模式下：
  - 不提供“图标右键隐藏”入口（主界面本就无图标）。
  - 可浏览隐藏空间，但“暴露到主页面”操作置灰并给固定提示。

### 非目标（本期不做）

- 不做云同步（仅保证后续可扩展）。
- 不做复杂密码策略（错误次数限制、冷却、找回流程等）。
- 不做跨页/跨文件夹隐藏空间分组管理（隐藏空间统一平铺）。
- 不做 URL 归一化（仅按原始字符串相等去重）。

## 2. 产品规则（已确认）

1. 开关打开逻辑：先弹“设置密码”对话框，输入并确认成功后才真正开启。  
   若用户取消，则回滚为关闭状态。
2. 密码规则（v1 简化）：
   - 长度 `>= 4`
   - 不限制字符类型
   - 不限制错误次数
   - 无冷却时间
3. 数据模型：图标从主列表“搬移”到 `hidden` 列表，不做软标记。
   - 若隐藏对象是文件夹，则将其内部站点展开为单图标写入 `hidden` 列表。
   - 隐藏空间仅存单图标，不保留文件夹元数据。
4. 批量操作作用域：仅隐藏空间内多选，不涉及跨页/跨文件夹语义。
5. “暴露到主页面”落位：
   - 统一按主桌面“最后一页末尾”追加平铺
   - 若当前页面容量不足，自动向右新增页面
   - 不尝试恢复原始位置/原始文件夹
6. 极简模式：
   - 隐藏空间数据保留、可浏览
   - “暴露到主页面”按钮置灰，并提示：  
     `极简模式下不可恢复到主页面，请切换为默认模式`
7. 关闭隐藏空间：
   - 先输入密码校验
   - 再展示“清空数据”确认
   - 确认后清空 `hidden` 列表 + 清空密码 hash + 关闭开关
   - 关闭后可再次重新设置密码开启

8. 去重规则：
   - 隐藏写入时按 URL 原始字符串去重（严格相等）。
   - 若 URL 重复，视为同一目标，新项自动跳过。
   - 不做命名冲突处理。
9. 批量隐藏反馈：v1 不额外展示“新增/去重统计”反馈。
10. 执行顺序：先计算待新增列表（含去重），再删除源项，避免中途状态闪烁。
11. 隐藏空间内浏览与编辑（与实现对齐）：
   - **浏览态（默认）**：点击图标打开链接；不显示批量工具条。
   - **编辑态**：点击顶部 `编辑` 进入；再点卡片为**选中/取消选中**（点一下选中，再点取消）；点 `完成` 退出编辑态并**清空选中**（不另弹确认）。
   - **全选 / 全部取消**：单个按钮切换文案；**仅在编辑态且列表非空**时显示。
   - **空列表**：`编辑` 置灰不可用；无全选按钮。
   - 批量删除确认文案**不带数量**（固定一句，如「确认删除所选隐藏图标吗？」）。
   - 批量暴露或删除后**保持编辑态**；删除成功后可清空当前选中以便继续操作，不改变「是否在编辑态」。
12. 密码相关弹框：输入框支持 **Enter** 提交（与「开启」「确认」按钮等价）。
13. 设置页生命周期：
   - 切换到**其他设置分类**或**关闭设置弹窗**时：隐藏空间面板视为收起；**退出编辑态并清空选中**；不保留「上次已展开」状态。
14. 首次隐藏文件夹提示：
   - 用户第一次执行“隐藏文件夹”时弹确认提示。
   - 提示确认后写入“已提示”标记，后续不再弹。
   - 可提供“重置隐藏文件夹提示”能力，但该入口仅在 dev 模式可见；release 正式版不显示。

## 3. 状态机（v1）

```ts
type HiddenSpaceStatus = "disabled" | "enabled";
```

- `disabled`：未启用；无密码；隐藏列表应为空。
- `enabled`：已启用；存在密码 hash；可进入隐藏空间。

约束：

- 不引入 `enabled-uninitialized` 中间态。  
  开启流程采用“先设密码再写 enabled”，避免脏状态。
- 每次“进入隐藏空间”“关闭隐藏空间”都要求密码校验。

---

## 4. 数据模型与存储

```ts
type HiddenSpacePreference = {
  enabled: boolean;
  passwordHash: string | null; // v1: sha256(plainText)
};
```

```ts
type HiddenItemsPayload = {
  /** 扁平化为站点图标项（`type: "site"`），与主网格类型兼容便于恢复 */
  items: SiteItem[];
};
```

建议新增存储键：

- `xallor_hidden_space_pref`
- `xallor_hidden_space_items`

读取规则（sanitize）：

- `enabled !== true` 视为关闭。
- `passwordHash` 非字符串视为 `null`。
- `items` 非法回退空数组。
- 若发现 `enabled = true` 但 `passwordHash = null`，自动修复为关闭并清空 items（防异常态）。

## 5. 关键交互流程

### 5.1 从右键隐藏图标

- 入口：图标/文件夹右键菜单统一显示 `隐藏`。
- 前置校验：
  - 若隐藏空间未开启：禁止执行，提示去设置中开启。
  - 若当前为极简模式：理论上无图标入口，不额外提供动作。
- 成功路径：
  - 若对象为图标：从当前主列表删除并追加到隐藏列表（末尾）
  - 若对象为文件夹：首次执行时先弹确认“文件夹将被删除且内部图标将移入隐藏空间”；确认后将文件夹内站点展开为单图标写入隐藏列表（按 URL 去重），再删除源文件夹
  - 执行顺序固定为“先计算新增列表，再删除源项”

### 5.2 开启隐藏空间

- 设置开关从 off -> on 时：
  1. 弹设置密码对话框（密码 + 确认密码）
  2. 校验通过后写入 `passwordHash` 并 `enabled=true`
  3. 关闭弹框
- 若取消或校验失败未提交：保持 `enabled=false`

### 5.3 浏览隐藏空间

- 入口：设置 `隐私与安全` 内「浏览隐藏空间」→ `进入`（仅在**隐藏空间已开启**时显示该行）。
- 进入前弹密码验证框；**Enter** 可提交验证。
- 验证成功后展开下方隐藏空间区域：
  - 布局为**图标网格**（与打开文件夹后的内部网格语义一致，便于用户对照主界面图标）。
  - 默认浏览态；点 `编辑` 进入编辑态后再批量操作。
- 收起：面板内 `收起`，或随设置分类切换 / 关闭设置而自动收起（见上文「2. 产品规则」第 13 条）。

### 5.4 关闭隐藏空间

- 设置开关从 on -> off 时：
  1. 先弹密码验证框
  2. 验证成功后弹“清空数据确认”
  3. 确认后执行清空并关闭
- 任一步取消：不改变当前状态

### 5.5 隐藏空间批量操作（编辑态内）

- 仅在**编辑态**展示批量区：`暴露到主页面`、`删除`；极简模式下仍可**删除**；`暴露到主页面` 置灰并给固定提示。
- 选中后支持批量暴露、批量删除（删除二次确认，文案不带数字）。
- v1 不额外展示批量操作统计反馈（如新增数/去重数）。
- 极简模式下“暴露到主页面”：
  - 按钮置灰
  - 显示固定提示：`极简模式下不可恢复到主页面，请切换为默认模式`

## 6. UI 结构（与当前实现对齐）

- **位置**：`设置` → `隐私与安全`。视觉与通用/外观等分组一致（圆角卡片、开关样式、分隔线）。
- **第一行（始终可见）**：
  - 标题：**隐藏空间**  
  - 说明：启用后可将内容隐藏到受密码保护的空间  
  - 右侧：开关（off → on 先走设密弹窗；on → off 先验密再清空确认）
- **仅当开关为 on 时显示**：
  - **浏览隐藏空间**：说明 + 右侧 `进入`（验密后展开下方区域）
  - **重置隐藏文件夹提示**（仅 dev / 本地开发 hostname 可见）：`重置` 用于再次触发首次隐藏文件夹确认
- **下方展开区**（验密进入面板后）：
  - 标题「隐藏空间」，工具栏：`全选/全部取消`（编辑态且非空）、`编辑/完成`、`收起`
  - 主体：文件夹内部同款的**图标网格**；浏览态点击打开链接；编辑态点选 + 底部批量按钮

交互文案建议（固定）：

- 未开启时右键提示：`隐藏空间未开启，请在设置中先开启该功能`
- 极简模式恢复提示：`极简模式下不可恢复到主页面，请切换为默认模式`
- 关闭确认：`关闭后将清空所有隐藏图标及密码，且不可恢复`
- 首次隐藏文件夹确认：`隐藏文件夹后，原文件夹会被删除，内部图标会按单图标形式移入隐藏空间，且不保留原文件夹结构。是否继续？`

## 8. 测试计划（必须）

遵循命名规范：`should_<expected_behavior>_when_<condition>`，并在测试函数前写简短注释。

### 偏好/存储层

- 开启时取消设置密码，应保持关闭。
- 开启成功后应写入 `enabled=true` 与 `passwordHash`。
- 关闭时密码验证通过 + 二次确认后，清空数据并关闭。
- 关闭流程任一步取消，不应改变当前状态。

### 组件交互

- 未开启时右键点「隐藏」，应失败并显示引导提示。
- 开启后右键隐藏，图标应从主列表移至隐藏列表。
- 首次隐藏文件夹应弹确认；确认后文件夹删除且内部图标迁入隐藏列表。
- 二次隐藏文件夹不应再弹首次确认提示。
- 浏览隐藏空间必须先验证密码。
- 批量删除应二次确认。
- 极简模式下“暴露到主页面”按钮置灰并显示固定提示。
- `重置隐藏文件夹提示`入口仅在 dev 模式可见，release 不显示。
- 隐藏空间未开启时，隐私页不展示「浏览」行与下方面板。
- 切换设置分类或关闭设置后，隐藏空间面板应收起（编辑态与选中清空）。

### 回归场景

- 批量暴露到主页面时，超容量自动新增页面。
- 批量暴露后不恢复原文件夹位置，仅平铺追加。

## 9. 分阶段实施顺序

1. 扩展 preference + storage（含 sanitize 与密码 hash 工具）。
2. 设置页接入：开启/关闭流程弹框与浏览入口。
3. 右键菜单接入“隐藏图标”与前置校验提示。
4. 新增隐藏空间浏览面板（多选、批量暴露、批量删除）。
5. 极简模式禁用“暴露到主页面”能力与固定提示。
6. 补齐测试并跑通 `typecheck` + `vitest`。

---

## 10. 后续扩展点（为云同步预留）

- 密码 hash 从 SHA-256 升级到带盐 KDF（如 PBKDF2/Argon2）。
- 增加失败次数与冷却策略。
- 隐藏空间与偏好进入“用户云配置”同步域。
- 支持密码重置策略（可选：重置即清空）。

---

## 11. 一句话决策

> 先以本地可用闭环为第一优先：开启强制设密、关闭强制验密并清空、主列表搬移进独立隐藏列表；设置内用「隐藏空间」命名，未开启时只显示开关；浏览采用文件夹同款网格，编辑态内多选批量操作；切换设置或关闭设置即收起并清空临时 UI 状态；极简模式可浏览与删除，禁止恢复到主页面。
