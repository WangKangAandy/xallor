# Interaction System Blueprint（交互系统蓝图）

## 1. 目标

将当前分散的交互规则（主题、命中面、模态阻断、键盘行为）收敛为统一蓝图，作为后续功能扩展与回归测试的上层约束。

适用范围：

- 新标签页主界面（搜索、网格、侧栏、弹层）
- 设置页与业务弹层
- 未来新增交互实体（组件、装饰层、编辑器等）

## 2. 系统分层

### 2.1 Visual Layer（视觉层）

- 主题运行时：亮/暗/系统偏好、token 生效链路。
- 层级规范：背景、主内容、浮层、模态、Portal 的 `z-index` 约定。

### 2.2 Hit Testing Layer（命中层）

- Pointer 命中与事件路由。
- 右键命中面契约（实体优先、最近命中优先、空白兜底）。
- 模态覆盖阻断（`data-ui-modal-overlay`）优先级。

### 2.3 Interaction Layer（交互层）

- 鼠标、键盘、触控行为一致性。
- 关闭策略（Esc、点外部、焦点转移）。
- 拖拽/整理模式与普通交互的互斥规则。

### 2.4 State Layer（状态层）

- 用户偏好状态（主题、布局、搜索引擎等）。
- 临时 UI 状态（弹层开关、选中态、hover/active）。
- 状态持久化边界与恢复策略。

## 3. 核心契约（当前版）

### 3.1 模态阻断优先

- 任何模态开启时，先满足阻断契约，再处理背景交互事件。
- 禁止“模态内点击穿透到背景网格”的隐式行为。

### 3.2 实体优先，空白兜底

- 命中实体区域，交给实体策略；
- 未命中实体，触发背景策略；
- 嵌套实体采用最近命中优先。

### 3.3 输入语义保留

- 输入与可编辑区域默认保留原生上下文能力（复制、粘贴、选择等）。
- 自定义菜单不得无条件覆盖输入语义。

### 3.4 可扩展标记驱动

- 新实体接入通过统一数据标记（如 `data-context-entity`），而非在主逻辑硬编码组件白名单。

## 5. 演进路线

### Phase A（已具备基础）

- 固化右键命中与模态阻断契约。
- 用回归测试兜住“穿透、抢占、边缘漏判”。

### Phase B（工程收敛）

- 抽象统一交互守卫（例如命中判定 helper / overlay 阻断 helper）。
- 清理分散在组件内的重复判定逻辑。

### Phase C（体验一致性）

- 补齐键盘行为矩阵（Tab、Esc、Enter、方向键）。
- 建立交互回归基线（关键路径 E2E + 组件层单测）。

## 6. 变更准入清单（建议）

新增交互能力前，至少回答：

1. 命中层归属是什么（实体/空白/模态）？
2. 是否影响输入语义或可访问性？
3. 是否会改变现有层级与穿透路径？
4. 是否补了对应回归测试？

若任一项不明确，不进入实现阶段。
