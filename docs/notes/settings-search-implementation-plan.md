# 设置搜索功能实施计划

## 1. 背景与目标

当前设置弹层顶部已有“搜索设置...”输入框，但仍是只读占位态，未接入真实检索与定位逻辑。随着设置项增多（主题、布局、搜索引擎、隐私等），仅靠手动切换一级栏目会提高定位成本。

本计划目标：

- 在不重构整体设置架构的前提下，交付可用的设置搜索能力。
- 让用户通过关键词快速定位到目标设置项（而非仅筛选一级栏目）。
- 与后续“一级/二级导航”方案保持兼容，避免重复造轮子。

## 2. 范围定义

### 2.1 本期（MVP）包含

- 搜索框支持输入、清空、基础关键词匹配。
- 命中后自动切换到对应一级栏目。
- 命中具体设置项时，主区自动滚动并定位到对应条目。
- 无结果时展示统一空态提示。

### 2.2 本期不包含

- 模糊搜索库（如 fuse.js）与复杂打分。
- 拼音首字母检索、同义词自动扩展（仅预留关键词位）。
- 完整“搜索结果下拉面板 + 键盘选择”复杂交互。

## 3. 方案设计

## 3.1 索引模型（稳定优先）

在设置模块内部维护一份静态索引（建议 `SETTINGS_SEARCH_INDEX`）：

- `sectionId`: 一级栏目（如 `general` / `appearance`）
- `anchorId`: 主区锚点（用于 `scrollIntoView`）
- `titleKey`: i18n 标题 key
- `descKey`: i18n 描述 key（可选）
- `keywords`: 额外关键词（中英混合）

说明：

- 不直接从 DOM 抽文本做检索，避免文案变动导致行为漂移。
- `keywords` 作为“文案变化缓冲层”，保证搜索稳定性。

## 3.2 匹配策略（先简单后增强）

- 输入归一化：`trim + toLowerCase`，并压缩多空格。
- 匹配逻辑：`title/desc/keywords` 的 `includes` 检测。
- 命中优先级：
  1) 精确关键词命中；
  2) 标题命中；
  3) 描述命中。

## 3.3 导航与定位行为

- 当 query 为空：保持当前行为，不主动跳转。
- 有命中：
  - 自动设置 `activeSection` 为首个命中项所在 section；
  - 若存在 `anchorId`，在渲染后滚动到目标条目并短暂高亮（可选）。
- 无命中：
  - 不切换 section；
  - 在主区展示“未找到相关设置”空态。

## 3.4 与二级导航方案兼容

本计划不阻塞 `settings-secondary-navigation-plan.md`，并复用字段：

- `sectionId` 可直接映射一级栏目展开。
- `anchorId` 可直接映射二级项高亮。

## 4. 实施步骤

### Phase A：最小可用

1. 将搜索框从只读改为受控输入。
2. 接入 `SETTINGS_SEARCH_INDEX` 和基础匹配函数。
3. 选择首条命中项驱动 `activeSection`。
4. 在目标设置行增加锚点 id/ref，实现滚动定位。
5. 加入“无结果”提示文案（中英文）。

### Phase B：可用性增强

1. 增加结果计数与结果摘要（如“找到 3 项”）。
2. 支持 `Esc` 清空输入。
3. （可选）命中项轻量高亮，帮助用户建立空间感。

### Phase C：导航联动

1. 与左侧二级导航对齐：搜索命中自动展开对应一级并高亮二级。
2. 补齐键盘导航（上下选择、Enter 定位）。

## 5. 文件改动建议

- `src/app/components/SettingsSpotlightModal.tsx`
  - 搜索状态、索引匹配、section 切换、锚点定位。
- `src/app/components/SettingsSpotlightModal.test.tsx`
  - 增加搜索命中、跨 section 跳转、无结果场景回归测试。
- `src/app/i18n/messages.ts`
  - 补充空态文案与（可选）结果数文案。

## 6. 测试策略

新增测试至少覆盖：

1. 输入“主题”可自动切到“外观”并看到主题设置。
2. 输入“打开方式”可定位到通用页对应条目。
3. 输入不存在关键词显示空态。
4. 清空输入后恢复普通浏览状态。

命名沿用约定：`should_<expected_behavior>_when_<condition>`。

## 7. 风险与规避

- 风险：文案频繁调整导致搜索不稳定。  
  规避：索引中维护稳定 `keywords`，避免完全依赖展示文案。
- 风险：滚动定位受异步渲染时序影响。  
  规避：基于 `requestAnimationFrame` 或布局 effect 在 section 切换后再定位。
- 风险：后续二级导航重复实现。  
  规避：本期提前固化 `sectionId + anchorId` 双键模型。

## 8. 验收标准

- 用户可通过搜索词在 1~2 次输入内定位常见设置项。
- 不引入设置弹层关闭、遮罩点击、主题切换等已有回归问题。
- 测试通过：`SettingsSpotlightModal` 新增搜索相关回归用例全部通过。
