# 首页三层视觉节奏改造计划

## 0) 目标与边界

- 目标：把首页视觉重心收敛成 3 层级：`搜索框 > 核心组件 > 普通入口`，解决“都在抢第一层注意力”的平铺感。
- 边界：本轮只改视觉层级与样式 token，不改核心交互逻辑（拖拽、分页、添加入口等）。

## 1) 建立三层设计 token（先定规则，再改组件）

- 在 `theme.css` 新增（或扩展）语义变量：
  - `--hero-*`（第一层搜索）
  - `--anchor-*`（第二层核心组件）
  - `--shortcut-*`（第三层普通站点）
- 每层至少定义：`bg / border / shadow / blur / radius / title-color / gap`。
- 输出一份简短对照表（写入 docs）：组件 -> 层级 -> 使用 token。

## 2) 第一层（主入口）改造：搜索框

- 目标：让搜索框成为页面“标题级主操作入口”。
- 调整方向：
  - 宽度上限增加（更接近主内容列宽）
  - 玻璃底更实（透明度降低）、阴影和模糊略增强
  - 上下留白增大，拉开与下方网格距离
  - 保持克制，不做高饱和强调色
- 验收：打开首页 1 秒内，视线第一落点稳定在搜索框。

## 3) 第二层（核心组件）改造：天气 + 文件夹

- 天气卡定位为“信息锚点”：
  - 保持较高权重（面积/对比/信息密度）
- 文件夹定位为“功能锚点”：
  - 相对天气降权（尺寸或对比下降其一）
  - 避免多个大文件夹并排同权抢焦点
- 验收：天气明显高于文件夹；文件夹高于普通站点。

## 4) 第三层（普通快捷入口）改造：单站点卡

- 统一小卡尺寸与节奏（同一行列视觉一致）
- 降低厚重感（更轻的底色、边框和阴影）
- 文字标签降权（颜色更浅、字重更轻）
- 强化组关系（排布更整齐，避免“每个都像主角”）
- 验收：普通入口成为“支持层”，不与天气/文件夹争主视觉。

## 5) 结构与复用（防回归）

- 基于已落地的 `gridTile/gridPanel` 体系继续推进：
  - 搜索框层级样式走独立 `hero` token
  - 网格卡片继续复用 `GridDesktopCardSurface`，只在 token 层做层级区分
- 禁止在业务组件里新增新的“手写玻璃配方”（避免再次分叉）。

## 6) 分阶段落地顺序（建议两次提交）

- 提交 A（基础设施）
  - 新增三层 token + 文档映射
  - 不改布局，仅替换现有样式引用
- 提交 B（视觉定标）
  - 搜索框尺寸/留白调优
  - 天气与文件夹权重差拉开
  - 普通站点入口降权
- 每次提交都跑：`typecheck + test + build`。

## 7) 验收清单（可截图对比）

- 视觉落点顺序：搜索框 > 天气 > 文件夹 > 普通站点
- 页面 100% 缩放下，无新增滚动条/遮挡
- 拖拽/合并/重命名/分页行为不回归
- 深浅主题（若开启）层级关系一致，不反转
