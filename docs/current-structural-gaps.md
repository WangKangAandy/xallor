# 当前结构性不足清单（开工整理模式前）

本文档聚焦“会导致后续反复补坑”的结构性问题，不讨论视觉细节。

---

## 1) 运行时真相与元数据双轨张力

- 当前已约定：运行时以 `items` 为真相，`widgetLayout` 作为元数据/策略。
- 风险：`widgetLayout.layout` 仍保留坐标语义，容易被误用为渲染驱动，导致双轨回潮。
- 建议：
  - 明确标注 `widgetLayout.layout` 在当前阶段不参与渲染。
  - 所有 item 变更统一走单入口（避免绕过同步逻辑）。

## 2) `DesktopGrid` 职责偏重

- `DesktopGrid` 同时承担 DnD、弹窗、文件夹、策略按钮等多职责。
- 风险：整理模式叠加多选/批量拖拽后，事件优先级容易冲突。
- 建议：
  - 新增整理模式会话层（`useArrangeSession`）承接状态机。
  - `DesktopGrid` 保持“渲染 + 事件转发”。

## 3) 跨页与跨容器操作缺统一命令层

- 现有按 pageId 分片管理是正确方向，但缺“批量操作命令层”。
- 风险：跨页拖拽、文件夹内外统一选择时出现状态漂移。
- 建议：
  - 统一命令：`toggleSelect / deleteSelected / moveSelected / setHomePage`。
  - 跨页操作只通过 reducer action，不在组件里散改。

## 4) 文件夹内外选择集尚未统一建模

- 目前文件夹内容通过 Portal 展示，外层网格是另一渲染树。
- 风险：后续“外层+文件夹内同时多选”易出现漏选/误删。
- 建议：
  - 使用复合 ID 体系（`item:{page}:{id}`、`folder-item:{page}:{folder}:{site}`）。
  - 选中集提升到整理模式会话全局。

## 5) 全局交互副作用需收敛

- 已加入小憩模式全局监听（双击、键鼠唤醒、全屏尝试）。
- 风险：全局监听与未来整理模式拖拽事件互相影响。
- 建议：
  - 按模式启停监听，保持最低权限。
  - 统一交互优先级：框选 > 批量拖拽 > 单项拖拽/resize。

## 6) 测试覆盖偏“局部”，缺“编排级”

- 已有函数级和组件级测试基础。
- 风险：整理模式核心问题发生在“跨组件编排路径”。
- 建议：
  - 为整理模式补状态机/会话层测试。
  - 每阶段至少一条“跨页 + 文件夹 + 批量操作”回归用例。

---

## 对整理模式的落地要求（前置门槛）

在进入大规模编码前，至少满足：

1. 明确“当前运行时真相 = `items`”且团队共识一致。  
2. 整理模式状态机独立，不把复杂分支直接塞回 `DesktopGrid`。  
3. 批量删除/移动统一走命令层，支持回归测试。  
4. 文档与代码阶段状态保持同步。
